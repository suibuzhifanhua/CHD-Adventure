## 一、游戏概述

**彩虹岛大冒险**（Rainbow Island Adventure）是《彩虹岛 Online》内置的大富翁式棋盘游戏。玩家在由 **75 个阶段**组成、总计约 **2898 格**的棋盘上投掷骰子前进，核心目标是：**用 100 个骰子走尽可能多的格子**。

| 属性 | 数值 |
|------|------|
| 棋盘总格数 | 2898 |
| 阶段数 | 75 |
| 每轮骰子数 | 100 |
| 骰子类型 | 2d6（两个六面骰子） |
| 幸运卡种类 | 22 种（核心可用） |
| 幸运卡持有上限 | 5 张 |

## 三、棋盘格子类型

根据 TILE_DATA 解包数据，棋盘上共有以下格子类型：

### 3.1 基础格子

| 类型 | 标识 | 效果 |
|------|:----:|------|
| **起点 (START)** | 🏁 | 每圈经过时获得额外骰子 |
| **终点 (END)** | 🏆 | 阶段/棋盘终点 |
| **普通格 (NORMAL)** | ⬜ | 无特殊效果，正常通行 |

### 3.2 事件格子

| 类型 | 标识 | 效果 |
|------|:----:|------|
| **幸运卡 (CARD)** | 🃏 | 踩中时按概率抽取幸运卡（~70%概率中卡） |
| **首领格 (BOSS)** | 👹 | 经过时**必定停留**，消耗 1 个骰子处理 |
| **强制停止 (STOP)** | ⛔ | 踩中时强制结束当前回合 |

### 3.3 传送格子

| 类型 | 标识 | 效果 |
|------|:----:|------|
| **管子捷径 (WARP)** | 🌀 | 踩中后传送到指定出口格，白嫖一段距离 |
| **大捷径 (WARP_BIG)** | 🌪️ | 更远距离传送 |
| **超级捷径 (WARP_SUPER)** | ⚡ | 超级传送 |
| **跳王捷径 (KING_WARP)** | 👑 | 特殊传送，可跨阶段跳跃 |

传送管（WARP）机制：踩中水管 IN 格后自动跳转到对应 OUT 格，传送链可有多段（IN → OUT → IN → OUT ...）。引擎使用 `followWarpChain()` 统一追踪嵌套传送链，所有传送点均经过完整链路展开，避免漏算跳跃距离。主动用前进卡精准踩水管 IN 是高级策略之一。

### 3.4 奖励格子

| 类型 | 标识 | 效果 |
|------|:----:|------|
| 冒险点数 (POINT) | 💰 | 直接获得冒险点数 |
| 评价点数 (GRADE) | ⭐ | 获得评价系统点数 |
| 超人点数 (SUPER) | 💪 | 获得超人系统奖励 |
| 金宝箱 (CHEST_GOLD) | 📦 | 稀有奖励 |
| 蓝宝箱 (CHEST_BLUE) | 💎 | 中等奖励 |
| 木宝箱 (CHEST_WOOD) | 🪵 | 普通奖励 |

### 3.5 事件/特殊格子

| 类型 | 标识 | 效果（推断） |
|------|:----:|------|
| 红色事件 (EVENT_RED) | ❗ | 强制前进 N 格（老虎机箭头牌） |
| 蓝色事件 (EVENT_BLUE) | 💙 | 触发随机蓝色事件 |
| 黄色事件 (EVENT_YELLOW) | 💛 | 强制后退 N 格（老虎机三角牌） |
| 召唤兽 (SUMMON) | 🐉 | 召唤兽相关奖励/事件 |
| 陷阱 (TRAP) | ⚠️ | 触发负面效果 |
| 通用事件 (EVENT) | ❓ | 通用随机事件 |

> **注**：带"(推断)"标记的格子类型在解包数据中未明确定义具体效果，根据格子分类名称合理推断。

### 3.6 ★ 强制移动格（v5 新增）

| 类型 | 标识 | 效果 | 默认步数 |
|------|:----:|------|:--------:|
| **强制后退 (MOVEBACK)** | 🔙 | 踩中后强制后退 N 格 | 3 格 |
| **强制前进 (MOVEFWD)** | 🔜 | 踩中后强制前进 N 格 | 4 格 |

**重要机制**：
- **EVENT_YELLOW**（黄色三角牌）自动解析为 MOVEBACK，**EVENT_RED**（老虎机箭头牌）自动解析为 MOVEFWD
- 强制移动后仍会触发后续效果：WARP 传送链、BOSS 停驻消耗骰子、卡片格抽卡
- 主引擎、快模拟、MCTS rollout、Worker 线程均完整支持此机制
- DOUBLE 触发的强制移动也会走完整处理流程

### 3.7 棋盘高级机制

| 机制 | 说明 |
|------|------|
| **传送链追踪** | `followWarpChain()` 统一处理多级嵌套传送（IN→OUT→IN→OUT...），所有传送到位计算均经完整链路展开 |
| **强制移动格** | MOVEBACK（向后推移）/ MOVEFWD（向前推移）踩中时强制移动指定步数，均需检查目标是否为 BOSS |
| **STOP 格** | 踩中时强制结束当前回合。若 STOP 格由 DOUBLE 触发到达，DOUBLE 投掷机会被 STOP 吞掉（互斥） |
| **卡片移动忽略 BOSS/STOP** | 使用前进卡/后退卡移动时，即使经过或落在 BOSS/STOP 格也不会触发停留/停止效果 |
| **DOUBLE 与 STOP 互斥** | DOUBLE 投掷结果若踩中 STOP 格，不会再获得额外 DOUBLE 机会 |

## 四、幸运卡系统

### 4.1 抽卡规则

- **卡池**：共 22 种核心卡片（前进卡 12 种 + 后退卡 3 种 + 倍数卡 6 种 + 跳阶段卡 1 种），采用**循环池机制**——不重复抽取，全部抽完后自动重置卡池重新洗牌。
- **抽卡触发**：踩中幸运卡格时，有约 70% 概率抽取一张卡片。
- **持有上限**：最多同时持有 5 张。超过上限时无法继续抽卡，必须消耗手牌腾出槽位。
- **等概率抽取**：所有卡片在池中等概率抽取。

> **注**：8 种"其他卡"（召唤首领、移动到指定格、额外骰子、护盾、陷阱卡、金币卡、事件卡）因游戏尚未设计完成，暂时禁用。

> **v5 策略转变**：后退卡不再是纯负资产。核心价值在于：
> 1. **以退为进**：后退后若落地为 WARP 格，触发传送链可实现净前进（>50 格大跳跃奖励 200 分）
> 2. **换卡策略**：后退至卡片格换取新卡，与前进卡的换卡逻辑统一评分
> 3. **跨阶段战术**：从阶段起点附近退到前一区域的 WARP 格，触发跨阶段传送（+60 分高级奖励）
> 4. **正向净收益**：WARP 链分析自动计算后退后的净位移，净收益 >0 时推荐使用

## 五、★ v5 深度 WARP 链分析系统

v5 引入了全新的深度 WARP 链分析引擎（`analyzeWarpChainDeep`），在所有决策路径中统一使用。

### 5.1 倒退卡 WARP 链分析（最大优化点）

分析每张后退卡落点的 WARP 链收益，自动计算：
- WARP 链总收益（每格 ×2 权重）
- 净位移（相对于原始位置的前进/后退）
- 卡片格奖励（落地卡片格 +100，每张途径卡片 +30）
- BOSS 惩罚（每经过一个 BOSS -50）
- 跨阶段后退加成（+60）
- 大跳跃奖励（净收益 >20 额外 ×2，>50 +200）

分析结果直接整合到"现在该做什么"主建议面板，净收益 >0 时以最高优先级推荐。

### 5.2 步数卡 WARP 格预判

对 12 个步数卡（+1 ~ +12）逐一分析：
- 落点是否为 WARP 格
- WARP 链后的最终位置
- 从最终位置下次掷骰再踩 WARP/卡片的概率
- 自动排序：优先显示 WARP 命中 + 下次高概率组合
- 三档推荐：⭐⭐⭐ 强烈推荐 / ⭐⭐ 换卡好 / ⭐ 可考虑

### 5.3 倍数卡 WARP 格预判

分析倍数卡激活后的掷骰范围中，各点数落点的 WARP 概率分布。

## 六、预测工具使用指南

### 6.1 工具功能概述

**rainbow-predictor-v4.html** 是一个完整的大富翁策略模拟与预测工具，基于蒙特卡洛方法，模拟 100 骰子的完整游戏过程，帮助玩家做出最优决策。

核心功能：
- **实时策略推荐**：根据当前格数、手牌库存和棋盘数据，给出即时行动建议。v5 整合后退卡 WARP 分析和 BOSS vs 倍数卡收益对比
- **MCTS 树搜索**：每步枚举所有合法动作（含后退卡），500 次 rollout 选最优，预期提升 5~15%
- **深度 WARP 链分析**：v5 算法——倒退卡/步数卡/倍数卡三重 WARP 预判
- **实时 MCTS 分析**：位置/手牌变化时自动异步计算最佳动作，1.5s 防抖，30s 缓存
- **蒙特卡洛模拟**：批量运行数百局模拟，对比多种策略的预期效果
- **操作记录**：开始/停止记录按键，自动保存掷骰子和用卡操作到 JSON 存档
- **完整棋盘建模**：75 阶段制、22 种卡片、DOUBLE 机制、传送链、BOSS 停驻、MOVEBACK/MOVEFWD 强制移动等
- **地图数据浏览器**：查看所有地图的格子类型分布

### 6.2 四种策略模式

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| **MCTS 树搜索** | 每步枚举所有合法动作（含后退卡），500 次 rollout 模拟取最终步数最优，超时 5s 退避启发式 | 追求理论最优步数 |
| **最优策略** | 基于多步前瞻（300 采样 × 6 步深度）、阶段自适应、BOSS 绕行、传送管主动踩、后退卡 WARP 链分析的综合决策引擎 | 日常使用，速度快 |
| **随机用卡** | 40% 概率随机使用手牌，模拟玩家无策略时的表现 | 作为对比基线 |
| **仅基础骰子** | 完全不使用卡片，仅依靠 2d6 投掷和 DOUBLE | 作为下限参考 |

四种模式在蒙特卡洛模拟中同步对比，产生"MCTS vs 基础"和"最优 vs 基础"的**提升百分比**。

### 6.3 蒙特卡洛模拟参数

### 6.4 统计指标解读

| 指标 | 含义 |
|------|------|
| **均值（格）** | 多次模拟的最终格数平均值，最核心的预测指标 |
| **中位数** | 50% 分位数，反映"典型"表现 |
| **最佳** | 所有模拟中的最高格数，反映运气上限 |

| **标准差** | 结果波动程度，越小越稳定 |
| **提升百分比** | 最优策略相对基础策略的均值提升比例 |
| **均值（阶段）** | 多次模拟的平均到达阶段数 |

### 6.5 卡片库存管理

- **添加卡片**：点击手牌区域空白格子，从弹窗中选择卡片
- **移除卡片**：点击已装备的卡片即可移除
- **爆卡警告**：卡槽 ≥ 4 张时，卡片显示脉冲动画提醒
- **持久化**：手牌和格数自动保存到浏览器 localStorage

### 6.6 模拟日志解读

每次蒙特卡洛完成后，在"最近模拟日志"面板展示最优策略最后一局的完整日志：

- 🃏 抽卡记录：显示抽到的卡片名称、当前手牌数量
- 🎯 使用记录：显示使用的卡片名称、移动轨迹
- 💡 决策记录：显示策略引擎的决策理由（含"激进"/"保守"标签）
- 👹 首领消耗：经过 BOSS 格时消耗骰子的记录

### 6.7 操作记录与存档

- **开始记录**：点击按钮后，自动记录每次掷骰子和用卡操作（含位置、骰子余量、手牌快照等）
- **停止记录**：点击按钮后，自动将操作历史序列化为 JSON 存档
- **自动保存**：优先使用 File System Access API 写入本地 `save\` 目录；不支持时回退到浏览器 Blob 下载
- **防重复**：位置不变时不重复写入记录，手动修改位置/增减手牌/切换策略/用卡均触发记录

### 6.8 实时 MCTS 分析面板

位置或手牌发生变化时，自动异步运行 MCTS 计算当前最佳动作：

| 特性 | 说明 |
|------|------|
| **防抖** | 1.5 秒内连续变化只触发一次计算 |
| **缓存** | 相同位置+手牌组合 30 秒内复用缓存结果 |
| **重入保护** | 上次计算未完成时不重复启动 |
| **超时退避** | MCTS 搜索 3s 超时自动退避到启发式策略 |
| **面板展示** | 金色边框面板，显示推荐动作、预期最终步数、备选动作列表 |

## 七、策略引擎架构

> v5 的策略引擎在 v4 的双引擎基础上，新增深度 WARP 链分析、后退卡统一评分、BOSS vs 倍数卡收益对比三大模块。

### 7.1 整体架构

```
用户输入（位置 + 手牌 + 骰子余量）
         │
         ├─── 手动触发 ──→ MCTS 树搜索 (mctsDecision)
         │                      │
         │                  超时5s? ──是──→ 启发式退避 (makeDecision)
         │                      │
         │                     否
         │                      │
         │                  返回最优动作
         │
         ├─── 自动触发 ──→ 实时 MCTS (runRealtimeMCTS)
         │                      │
         │                  防抖1.5s + 缓存30s
         │                      │
```

### 7.2 MCTS 树搜索引擎

MCTS（蒙特卡洛树搜索）是核心决策引擎，在每步决策时枚举所有合法动作（v5 含后退卡），通过大量 rollout 模拟找到理论最优选择。

| 参数 | 默认值 | 说明 |
|------|:------:|------|
| rollout 次数 | 500 | 每个动作的模拟局数；动作 >8 时降为 200 |
| 超时时间 | 5s | 超时后退避到启发式 `makeDecision` |
| 动作枚举 | 全量 | 所有手牌（含后退卡） + "不使用卡片仅投骰子" |
| 模拟引擎 | `fastSimulateRun` | 轻量版仿真，支持 MOVEBACK/MOVEFWD/后退卡，无日志输出 |

**搜索流程**：
1. 深度克隆当前游戏状态（位置、骰子、手牌、倍数状态等）
2. 枚举所有合法动作（每张可用手牌 + "不使用"选项）
3. 每个动作执行 500 次快速 rollout，记录每次最终步数
4. 取各动作平均最终步数，选最高者作为推荐
5. 返回最优动作 + 所有动作评分列表（降序）

**状态克隆**：每次 rollout 前深拷贝状态对象，确保 rollout 间完全隔离、无状态污染。

### 7.3 启发式退避引擎

当 MCTS 超时或被禁用时，退避到启发式 `makeDecision` 引擎。

#### 三阶段自适应策略

策略引擎根据**剩余骰子数**动态调整决策权重：

| 阶段 | 剩余骰子 | 核心目标 | 决策系数 |
|------|:--------:|----------|:--------:|
| 保守期 | ≥ 40 | 攒卡为主，高阈值使用 | ×0.7 |
| 平衡期 | 15 ~ 39 | 正常运营，稳定推进 | ×1.0 |
| 激进期 | ≤ 15 | 最大化爆发，无条件用卡 | ×1.5 |

#### 多步前瞻

在使用步数卡/后退卡前，执行 300 次采样 × 6 步深度的快速蒙特卡洛前瞻：
- **路径A（不用卡）**：正常投掷 6 次 → 期望位置
- **路径B（用卡）**：用卡后投掷 6 次 → 期望位置
- 路径B 显著更优时推荐用卡

#### 核心机制

| 机制 | 说明 |
|------|------|
| **传送管主动踩** | 扫描前方 1-12 格 WARP IN，前进卡精准落在 IN 格上触发传送链 |
| **BOSS 绕行** | 扫描前方 12 格 BOSS 格，步数卡跳过后节省 1 骰子停留消耗 |
| **以卡换卡** | 步数卡优先落到幸运卡格，赚取额外抽卡机会；后退卡也参与换卡评分 |
| **倍数卡一鱼两吃** | BOSS 窗口内使用倍数卡，同时跳过 BOSS + 概率落到卡片格 |
| **★ 以退为进** | 后退卡 WARP 链分析：后退落点为 WARP 格时，传送链可实现净前进 |
| **★ BOSS vs 倍数卡收益对比** | 在 BOSS 前方时，自动对比步数卡绕行 vs 倍数卡换卡的收益排名 |

### 7.4 统一卡牌效果引擎（v5 重构）

## 十、常见问题

**Q：为什么模拟结果和我实际游戏的步数有差异？**

A：模拟基于 TILE_DATA 中的静态棋盘数据建模，已实现所有核心机制（2d6、DOUBLE、卡片、传送、BOSS 停驻、MOVEBACK/MOVEFWD），但由于实际游戏中存在服务器端随机种子、实时 PvP 干扰、事件选择等人为因素，模拟结果与实际可能有小幅偏差。

**Q：后退卡现在真的能用了吗？**

A：v5 策略转变后，后退卡不再是"永遠不用"。当后退落点为 WARP 格（尤其跨阶段 WARP）时，净收益可能为正。工具会自动分析每张后退卡的 WARP 链收益，净位移 >0 时推荐使用。但纯后退无 WARP 时仍不推荐。

**Q：模拟 2000 次会不会很慢？**

A：2000 次 × 4 策略 = 8000 局模拟，使用异步 setTimeout 执行，通常在 5-15 秒内完成。建议日常使用 100 次获得快速参考，深度分析时使用 500-1000 次。

**Q：跳阶段卡应该什么时候用？**

A：工具会自动计算"现在用"和"留到阶段开头用"的步数最大化对比。有时保留到下一阶段开头可以节省更多格数。会在分析面板中明确提示是否建议留着。
